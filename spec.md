# spec.md — RESTful Roguelike Game (Flask-Based)

## 1. Overview

### 1.1 Purpose

Design and implement a minimalist roguelike game inspired by *Rogue*, accessible entirely via a REST API. The system should support both human and AI agents interacting programmatically, enabling use cases such as bots, social media integrations, or unconventional interfaces.

### 1.2 Design Goals

* Stateless-friendly API with optional lightweight persistence
* Deterministic and reproducible game states (seed-based)
* Simple but extensible game mechanics
* Minimal infrastructure (Flask + in-memory or lightweight DB)
* AI-agent friendly (clear schemas, predictable responses)
* Turn-based interaction model

---

## 2. Core Concepts

### 2.1 Game Model

* Grid-based dungeon (2D array)
* Turn-based system
* Single-player sessions
* Procedurally generated levels
* Fog-of-war (optional toggle)

### 2.2 Entities

* Player
* Enemies (basic AI)
* Items (weapons, potions, etc.)
* Tiles (floor, wall, stairs)

### 2.3 Game Loop

1. Client sends action
2. Server validates action
3. Game state updates
4. Enemies take turns
5. Response returned with updated state

---

## 3. Architecture

### 3.1 Tech Stack

* **Backend**: Flask (Python)
* **Data Storage**:

  * Default: in-memory dictionary
  * Optional: SQLite (via SQLAlchemy or lightweight ORM)
* **Serialization**: JSON
* **State Management**: Session-based via game_id

## 4. API Specification

### 4.1 Base URL

```
/api/v1
```

---

### 4.2 Create New Game

**POST /games**

#### Request Body

```json
{
  "seed": 12345,
  "player_name": "optional"
}
```
* Seed is optional, generated by system if not provided

#### Response

```json
{
  "game_id": "uuid",
  "state": { ...initial_state }
}
```

---

### 4.3 Get Game State

**GET /games/{game_id}**

#### Response

```json
{
  "game_id": "uuid",
  "turn": 5,
  "player": { ... },
  "visible_map": [ ... ],
  "log": [ "You moved north." ],
  "status": "ongoing"
}
```

---

### 4.4 Perform Action

**POST /games/{game_id}/action**

#### Request Body

```json
{
  "action": "move",
  "direction": "north"
}
```

#### Supported Actions

* `move` (north, south, east, west)
* `attack`
* `pickup`
* `use_item`
* `descend`
* `wait`
* `remove_armor`
* `don_armor`
* `remove_ring`
* `don_ring`
* `switch_weapon`

#### Response

```json
{
  "success": true,
  "events": [
    "You hit the goblin.",
    "Goblin dies."
  ],
  "state": { ...updated_state }
}
```

---

### 4.5 Delete Game

**DELETE /games/{game_id}**

#### Response

```json
{
  "deleted": true
}
```

## 6. Game Mechanics

### 6.1 Movement

* Cardinal directions only
* Collision detection with walls
* Moving into enemy triggers combat

### 6.2 Combat

* Turn-based
* Simple formula:

  ```
  damage = max(0, attacker.attack - defender.defense)
  ```
* Death removes entity

### 6.3 Enemy AI

* Based on the enemy types from the original Rogue game
* Basic:

  * If adjacent → attack
  * Else → random move or move toward player (optional)

### 6.4 Items

* Types:

  * Potion (heal)
  * Weapon (increase attack)
  * Armor
  * Rings (offer status effects)
  * Gold
* Inventory limit: configurable
* All items inspired by the original Rogue game

### 6.5 Level Progression

* Stairs (`>`) lead to next level
* New map generated with increasing difficulty
